Access Logging Notice

Access to this FHIR server is logged for statistical and accounting purposes. Each request is recorded with the following metadata:

  • FHIR operation type (e.g. read, search, create, update)
  • Resource type accessed (e.g. Patient, Encounter, Observation)
  • Client IP address and forwarded address (if behind a proxy)
  • HTTP User-Agent string identifying the client software
  • Request timestamp and processing duration
  • Query parameters

Resource content is not logged. No clinical data, patient identifiers, or FHIR resource bodies are captured in the access log.

These logs are used for usage analysis and governance reporting. If you have questions about data handling, please contact the server administrator.
